# OSLA Test Project

Небольшой Django + DRF сервис для трекинга объектов строительства. Репозиторий используется как база для тестового задания.

## Быстрый старт
1. Создайте виртуальное окружение: `python3 -m venv .venv` и активируйте его.
2. Установите зависимости: `pip install -r requirements.txt`.
3. Примените миграции: `python manage.py migrate`.
4. Запустите сервер: `python manage.py runserver 0.0.0.0:8000`.
5. Базовый REST API доступен по `/building/` (список/создание) и `/building/<id>/` (детальный просмотр).

## Тестовое задание
Нужно расширить систему объектов строительства так, чтобы один объект мог вестись сразу несколькими организациями.

### Что требуется
- Спроектировать и реализовать сущность организаций. Для каждой организации нужно хранить название, количество сотрудников и уникальный идентификатор (например, `external_id`).
- Обеспечить возможность связать один объект строительства с несколькими организациями (и при желании — несколько объектов с одной и той же организацией).
- Обновить API так, чтобы:
  - В ответах по объектам возвращался список привязанных организаций.
  - Можно было добавлять/обновлять/удалять информацию об организациях и их связях с объектом (nested-подход или отдельные эндпоинты — на ваше усмотрение, главное, чтобы сценарии были покрыты).
- Добавить все необходимые миграции, валидацию данных и описание того, как пользоваться новой функциональностью (короткий раздел в README или swagger/schema).
- Добавить фильтры, необходимы для получения определенных объектов/организаций

### Дополнительные задания
1. **Покрытие тестами.** Напишите unit/интеграционные тесты для ключевых сценариев (модели, сериализаторы, API). Покажите, как запускать тесты.
2. **Docker/production ready.** Подготовьте `Dockerfile` и, при необходимости, `docker-compose.yml`, чтобы сервис можно было запустить одной командой. Продумайте настройки для продакшена (переменные окружения, `collectstatic`, health-check, gunicorn и т.д.).

## Критерии приёма

- README обновлён так, чтобы по нему можно было быстро поднять проект и воспроизвести реализацию.
- Для дополнительных заданий — наличие инструкций по запуску тестов и docker-контейнеров.
